# neo DESIGN.md

版本基准：0.9.2  
最后整理：2026-06-03  
适用范围：当前 `neo` 桌面端源码、浏览器预览、桌宠窗口和本地服务端。

## 1. 产品定位

neo 是一个本地运行的桌面 AI 工作台。它不是单一模型，而是一个可配置供应商、模型、工具、文件工作区和桌宠交互的本地外壳。

核心目标：

- 像 Codex 一样安静、清晰、可长期工作的界面。
- 支持用户自带 API Key，切换国内外主流供应商和模型。
- 允许 AI 在明确授权范围内读取、生成和整理本地工作区文件。
- 用桌宠作为轻量陪伴入口，但不让桌宠功能喧宾夺主。
- 默认状态尽量安全，危险能力必须明确授权。

非目标：

- 不把 neo 包装成某个底层模型。
- 不默认控制整台电脑。
- 不把文件保存到模糊的沙箱位置。
- 不用脚本套脚本实现核心定时功能。
- 不用营销落地页式界面代替真实工作台。

## 2. 默认外观

当前默认外观必须固定为 `codex-v1`。这是 0.9.2 的默认视觉基线，后续打包和安装后也应保持一致。

默认外观参数：

```text
appearancePreset: codex-v1
globalOpacity: 1
glassBlur: 20
leftColor: #eef0f4
leftOpacity: 0.9
centerColor: #ffffff
centerOpacity: 0.98
rightColor: #f7f7f5
rightOpacity: 0.9
fontFamily: -apple-system, BlinkMacSystemFont, system-ui, sans-serif
```

视觉原则：

- 整体参考 Codex 桌面端：浅色、低噪音、半透明、轻毛玻璃。
- 主工作区保持大留白和低装饰，不做深色大渐变背景。
- 左侧栏、中央会话区、右侧工作台要像一个整体应用，而不是多个浮动卡片拼接。
- 按钮、tab、列表项保持简洁图标和短文字。
- 设置页图标使用简约线性图标，图标和文字间距参考 Codex 设置侧栏。
- 不使用强烈紫色虚线、过重描边或会干扰桌宠的拖拽装饰。

Stitch 适配原则：

- 可以借鉴 Stitch 设计稿，但不能丢掉 neo 当前三栏工作台、设置页和桌宠逻辑。
- `Luminous` 浅色稿更接近 neo 默认方向，可吸收玻璃侧栏、底部输入胶囊、右侧状态卡片和过程时间线。
- `Neural` 暗色终端稿只适合局部使用：终端输出、工具日志、运行状态，不作为整机默认主题。
- 不把 Stitch 里的外部品牌名、演示模型名、营销式顶部入口照搬进产品。
- 新视觉必须继续使用 `codex-v1` 作为默认外观基线，打包后也应保持一致。

## 3. 主界面布局

主界面三栏：

- 左侧：导航、定时任务、最近对话、设置入口、版本号。
- 中央：对话消息和输入框，是第一优先级区域。
- 右侧：工作台，包括任务、成果、文件、终端。

顶部标题栏：

- 左侧按钮控制左栏。
- 中间显示 `neo`。
- 右侧按钮控制工作台。
- Electron 系统菜单必须中文化。

右侧工作台：

- `任务`：显示当前任务状态、过程记录、工具调用。
- `成果`：显示上传、生成、下载、清洗后的文件。
- `文件`：浏览工作区、打开和保存文本文件。
- `终端`：仅用于用户手动命令，运行前必须确认。

## 4. 设置页设计

设置页打开后占用整个应用尺寸，左侧为 tab，右侧为当前设置内容。不要再用窄弹窗式设置。

tab 顺序和命名：

1. 模型
2. 供应商库
3. 助手设置
4. 外观
5. 桌宠
6. 技能库
7. 记忆
8. 环境检测
9. 关于
10. 归档

设计要求：

- tab 在左侧，图标与文字横向排列。
- 每个 tab 图标要与文字含义相关。
- 设置页右上角不要单独放关闭按钮，使用左侧“返回应用”。
- “检查更新”和“环境检测”必须是设置里的独立 tab，不放在右上角。
- “关于”页只显示 logo、名称、版本号和检查更新入口。
- “环境检测”页复用环境检测浮层的完整内容，并支持“不再提示”。

## 5. 模型和供应商

模型 tab 的目标是清晰可用，不是塞满信息。

模型页结构：

- 顶部状态条：当前对话、自动切换、连接状态。
- 左侧供应商列表：只显示是否可用。
- 右侧当前模型配置、选中供应商编辑、自动切换。
- 供应商列表避免显示过多 base URL 和模型数量，减少拥挤。

内置供应商原则：

- 只保留当前最好用、能按 API Key 直接切换使用的供应商。
- 旧的、重复的、已废弃或不可稳定使用的供应商应移出默认列表。
- API Key 不内置，用户本地配置保存在本机状态中。
- `neo-local` 作为本地演示保留，用于无 Key 状态下测试 UI。

当前内置协议：

- `mock`
- `openai`
- `anthropic`
- `gemini`

模型路由：

- 默认开启自动切换。
- 文本默认走 DeepSeek。
- 图片默认走 Qwen 视觉模型。
- 当用户问“你是什么模型”时，回答应说明当前供应商和模型，而不是只说 neo。

## 6. 助手设置

“助手设置”用于控制 neo 的助手人格、头像、角色和输出风格。

头像规则：

- 默认头像使用项目 logo：`/assets/logo.png`。
- 用户可以在主窗口助手设置里更换头像。
- 用户也可以通过对话上传图片并明确要求设置头像。
- 桌宠右键菜单不再提供“换头像”，避免与主设置重复。

命名原则：

- “助手设置”比“智能体选项”更适合当前产品：更自然、更贴近普通用户。
- 如果后续增加多智能体编排，再考虑使用“智能体”命名。

## 7. 桌宠设计

桌宠是 neo 的轻量陪伴入口，不是主工作区替代品。

桌宠能力：

- 点击头像打开/关闭桌宠聊天。
- 拖动头像移动桌宠位置。
- 拖文件到聊天窗口：作为附件加入对话。
- 拖文件到头像：触发“喂食”反馈，并把文件加入附件。
- 生成中发送按钮变成停止按钮，可以中断桌宠回答。
- 支持 Petdex 精灵图桌宠，包含待机、思考、执行、完成、错误预览。
- 支持桌面嗅探反馈，但敏感文件名需要避开明示。

桌宠右键菜单：

- 打开主窗口
- 安静模式
- 隐藏桌宠
- 退出 neo

桌宠视觉要求：

- 对话框不能被窗口边界裁切。
- 拖拽提示不能出现干扰视觉的紫色虚线。
- 头像、Petdex 精灵和聊天框尺寸要根据内容动态适配。
- 头像更换和桌宠选择必须区分：头像是助手形象，Petdex 是桌宠皮肤。

## 8. 本地工具和权限

neo 必须遵循最小权限原则。

默认权限：

```text
agentTools: false
fileRead: true
fileWrite: false
web: false
desktop: false
command: false
```

权限分组：

- `fileRead`：列出、读取、搜索工作区文件。
- `fileWrite`：写入、生成、导出、清洗工作区文件。
- `web`：搜索网页、读取网页、下载网页文件。
- `desktop`：打开网页、本地文件、本机应用、系统通知。
- `command`：运行本地命令。

约束：

- 前端开启工具时必须弹出授权确认。
- 服务端必须再次按授权过滤工具列表。
- 工具真正执行前还要二次检查权限。
- 本地命令工具默认关闭。
- 工作台终端即使是用户手动输入，也要运行前确认。
- Windows 或 macOS 系统操控必须提前获得明确授权。

## 9. 工作区和文件路径

工作区是 neo 文件能力的边界。

桌面端默认工作区：

```text
~/Documents/neo Workspace
```

桌面测试模式工作区：

```text
outputs/desktop-test/workspace
```

路径展示规则：

- 任何生成、上传、下载、清洗文件都显示“工作区内路径”。
- 成果区同时提供完整本地路径 title，方便定位。
- 文件浏览器显示“工作区根目录”而不是单独一个 `.`。
- 服务端 `resolveWorkspacePath` 必须阻止越过工作区。
- 附件默认保存到 `neo Attachments/YYYY-MM-DD/`。

成果区规则：

- 上传文件、生成文件、网页下载、表格清洗都进入成果区。
- 每个成果保留打开和定位按钮。
- 文件路径必须能通过常规系统方式找到，不能显示互相矛盾的沙箱/本地路径。

## 10. 定时任务

定时任务必须使用 neo 内置调度，不使用外部脚本套脚本。

支持表达式：

```text
daily HH:MM
weekly N HH:MM
hourly
interval Nm
interval Nh
```

设计规则：

- 服务端调度引擎在 `server/scheduler.mjs`。
- 任务持久化为 `schedules.json` 或用户数据目录下的 `schedules.json`。
- 任务标记 `engine: native`。
- 运行结果默认写入工作区文件。
- 定时任务启用工具时，只默认允许工作区文件读写，不默认允许打开应用或运行命令。
- UI 文案使用“neo 内置定时”，避免用户误以为依赖独立脚本。

## 11. 环境检测和更新

环境检测：

- 首次运行或缺少关键环境时提示。
- 用户可勾选“不再提示”。
- 仍可在设置的“环境检测”tab 手动查看和重新检测。
- 一键补齐环境前必须确认。

更新：

- “关于”页显示 logo、`neo v0.9.2` 和检查更新。
- 桌面测试模式不检查也不安装更新。
- 发布页入口保留在中文系统菜单“帮助”中。

## 12. 文本输出

输出展示要干净、可读。

展示层清理：

- 移除零宽字符。
- 过滤单独成行的 `--`、`---` 等脏分隔符。
- 清理多余行尾空格。
- 压缩过多连续空行。
- 代码块内部不做清理，避免破坏文件内容。

注意：

- 清理只用于可见聊天文本。
- 自动保存的文件块不应被展示层清理影响。

## 13. 工程结构

关键文件：

```text
public/index.html      主界面和设置页结构
public/styles.css      主界面视觉系统
public/app.js          主应用状态、聊天、设置、工作台、工具授权
public/pet.html        桌宠窗口结构
public/pet.css         桌宠窗口样式
public/pet.js          桌宠行为、聊天、拖拽、Petdex
electron/main.mjs      Electron 主进程、窗口、托盘、菜单、桌宠窗口
electron/pet-preload.cjs 桌宠预加载桥
server.mjs             本地 HTTP 服务入口
server/routes.mjs      API 路由
server/api.mjs         模型 API 调用
server/sse.mjs         流式输出和工具循环
server/tools.mjs       本地工具实现和权限闸门
server/scheduler.mjs   内置定时任务
server/environment.mjs 环境检测和安装命令
```

状态存储：

- 浏览器预览主要使用 `localStorage`。
- 桌面端使用 `appStatePath` 持久化应用状态。
- 工作区路径存储在 Electron userData 的 `workspace.json`。
- 桌宠设置存储在 Electron userData 的 `pet-settings.json`。

## 14. 后续改动边界

必须保持：

- 默认外观 `codex-v1`。
- 默认工具关闭。
- 默认头像 logo。
- 设置页左侧 tab。
- 关于页包含版本号和检查更新。
- 环境检测在设置 tab 内可访问。
- 桌宠停止按钮。
- 成果区路径可定位。
- 定时任务使用内置调度。

避免：

- 重新引入默认高权限。
- 在桌宠右键重新放“换头像”。
- 把供应商列表做得过度拥挤。
- 用外部提醒脚本替代内置 scheduler。
- 在打包版中回退到旧外观。
- 在 UI 内显示多余调试符号。

新增功能时优先考虑：

- 是否符合最小权限。
- 是否能在不打包的本地测试中验证。
- 是否会影响已确认的默认外观。
- 是否会让普通用户更容易理解，而不是增加设置复杂度。
