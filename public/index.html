neo 1.8.0 当前版本 · 操作日志（护栏第一步）
  • 新增「操作日志」：工作台多了一个「日志」标签，把 neo 每次调用工具（读写文件、运行命令、联网、生成文档等）的痕迹列清楚——工具名、成功/失败、耗时、产出文件路径。
  • 只记元数据、绝不记录正文内容（文件内容、网页正文、命令输出都不入库），全部留在本地、永不外发，随时可一键清空。
  • 联网更安全：抓取网页时改为手动跟随跳转，每一跳的目标都重新做内网/本地地址校验，堵住"公网链接 302 跳到内网"的绕过。
  • 这是「带护栏的自主代理」的第一步——先让每一步操作可被看见、可审计，后续再做任务级授权与全局急停。
neo 1.7.1 透视 / 分组汇总
  • 新增「透视 / 分组汇总」（pivot_table）：按维度分组对数值列做求和 / 平均 / 计数 / 最大 / 最小，结果另存为新的 Excel（或 CSV）。
  • 支持交叉透视表：设一个"透视列"，行 × 该列的不同取值生成矩阵（如"部门 × 月份"汇总金额）。
  • 常见用法："按部门按月统计销售额""各地区订单数""按项目汇总成本"——不用手写公式或拖透视表。
neo 1.7.0 数据表格智能（体检 + 出图）
  • 新增「数据质量体检」（check_table）：对 Excel / CSV 表格逐列体检——每列缺失率、主类型与类型一致性、完全重复行、数值异常值（IQR）、日期格式混用、单元格首尾空格、高频取值。
  • 拿到一份脏表先体检再清洗：「表格处理」「财务表格」技能会先用它找出问题，再决定怎么修。
  • 会直接给一份排好版的 Markdown 报告，可一键存成文档或 PDF。
  • 新增「表格出图」（chart_from_table）：从 Excel / CSV 直接画折线 / 柱状 / 饼图，选标签列和数值列即可，默认输出带文字、可缩放的 SVG（也可出 PNG）。
  • 纯本地完成、不依赖 Python、不联网、不需要额外模型。
neo 1.6.0 问我的文件（本地知识库）
  • 新增「问我的文件」：跨整个工作区按关键词检索，能搜进 Excel / Word / PDF / PPT 的内容（不只纯文本），找到最相关的文件再读出来回答并给出处。
  • 「本地文件助手」技能现在会先检索、再读取、带来源回答——不用你手动指明在哪个文件。
  • 检索更准：命中文件名、标题/首段、完整短语都会加权，长文档不再靠堆词频霸榜；结果片段对命中词做【】高亮，必要时给两段。
  • 记忆按工作区隔离：标记为"当前工作区"的记忆只在对应工作区里生效与注入，换项目不会串记忆；记忆列表会标出"其他工作区"的条目。
  • 关键词检索，本地完成、无需联网、不依赖额外模型。
neo 1.5.0 办公补缺（PDF / 模板 / OCR / md→Word）
  • 新增 PDF 导出：用 create_pdf_file 把 Markdown 或 HTML 渲染成真实 .pdf 文件并回读校验（如周报、说明文档一键出 PDF）。
  • 「文档阅读」技能现可生成 Word / PPT / PDF 三种文档。
  • 新增办公模板库（get_template）：周报 / 会议纪要 / 简历 / 项目计划 / 工作总结，取骨架填好直接出文档。
  • 新增读图片 / OCR（read_image_file）：读工作区图片或扫描件交给视觉模型读字、看图（需支持图片的模型）。
  • create_word_file 支持直接传 Markdown 自动转 Word（标题/列表/表格都转），同一份 Markdown 可出 PDF 也可出 Word。
  • PDF 用桌面端排版引擎渲染，质量高、无需额外依赖；浏览器预览模式暂不支持。
neo 1.4.0 看屏幕
  • 新增「看屏幕」技能：模型可截取当前屏幕画面来读取界面、定位元素、给操作指导（只读，不点击）。
  • 独立的「截取屏幕画面」权限，默认关闭；屏幕画面会发送给所选模型供应商，授权时明确提示。
  • 需桌面应用 + 支持图片输入的模型（如 Qwen-VL / GPT-4o 系 / Claude / Gemini）；截图自动作为图片喂给模型。
  • 修复 Claude 多轮工具调用时工具结果被降级成文本的问题。
neo 1.3.0 系统/软件设置脚本操控
  • 新增「系统与软件设置」技能：模型可按平台生成 AppleScript（mac）/ PowerShell（win）/ Shell 脚本来调整系统或软件设置。
  • 独立的「运行系统/软件设置脚本」权限，默认关闭；每条脚本运行前都在确认条上显示人话目的，由你单独放行。
  • 高危脚本（删盘、格式化、关机、fork 炸弹等）即使已授权也会被直接拦截，不进入执行。
neo 1.2.0 信息搜集闭环
  • 新增「信息搜集」技能：围绕一个主题多轮检索、批量读取来源，整理成带「## 来源」引用的 Markdown 简报落地工作区 Research 目录。
  • 新增 read_web_pages 批量读网页工具：一次并行读取多个来源正文，避免逐个读触达工具轮次上限。
  • 网页正文提取升级：优先抓取 <article>/<main> 主内容，剥离导航 / 页脚 / 侧栏噪音，简报更干净、更省上下文。
  • 搜索新增 site 限定站点与时效过滤参数（best-effort），中文 / 英文检索都能更精准。
  • 新增 save_research_report 工具：把搜集结果写成统一结构的 Markdown 简报，自动生成去重、带取用日期的来源清单。
  • 流式输出更顺滑：逐字平滑揭示 + 打字光标，不再一跳一跳；阅读历史时不会被强行拽到底部。
neo 1.1.0 稳定性加固
  • 修复网页抓取与搜索的中文乱码：按 HTTP 头、BOM 和 <meta> 自动识别 GBK/GB2312/GB18030 等编码再解码。
  • 搜索新增 Bing 国内站兜底，中文关键词优先走 Bing，英文仍优先 DuckDuckGo；并加入结果缓存与瞬时失败自动重试。
  • 流式输出更稳：正文中途卡死会按空闲超时自动停止；上游限流 / 欠费 / 模型不存在等流内报错会如实提示，不再只给空回复。
  • 抓取改用真实浏览器标识，降低被搜索引擎拦截返回空页的概率。
  • 生成的 Excel / Word / PPT 加强校验：空表会判为失败，特殊字符（&、<、>）转义有回归测试保障。
  • 安全加固：网页读取 / 下载拒绝访问本地与内网地址（含云元数据），防止被诱导请求内网。
  • 超长对话增加服务端兜底截断，保留 system 指令和最近对话，避免直接触发上游上下文超限报错。
neo 1.0.0 首个正式版
  • 新增浅色主题：设置 → 外观一键切换，界面与电光边框、粒子等动效整体换色，浅色动效采用晨雾靛紫（紫 + 蓝）配色。
  • Windows 窗口与 mac 对齐：去掉系统标题栏和菜单栏，改为透明无边框悬浮窗口，右上角内置最小化 / 最大化 / 关闭按钮。
  • 桌宠改为默认关闭，需要时在设置 → 桌宠里开启，托盘菜单的显示 / 隐藏与该开关保持同步。
  • 一键补齐环境强化：自动复检安装进度、失败汇总、支持国内镜像加速，修复新版 macOS pip 被 PEP 668 拦截的问题。
  • Windows 一键安装修复 PowerShell 启动报错，UAC 被拒绝时自动降级继续，缺少 winget 时自动补装。
neo 0.9.9 Living Assistant 界面
  • 重构主界面为 800×600 Living Assistant Lens 胶囊窗口，取消外层页面背景，让软件本体直接成为动态舱体。
  • 新增舱体鼠标跟随高光、呼吸光、轨道环和双层电流边框动效。
  • 能力入口改为右上角半隐藏胶囊，鼠标悬停或聚焦时展开历史、工作台、模型和设置。
  • 助手回复改为纯文字显示，去掉消息背景、模型提示行和普通回复里的模型自报。
  • 移除镜头界面的品牌标识、欢迎文案、输入框提示字和上下文状态标签，让画面更接近沉浸式助手。
neo 0.9.8.1 Kimi 与模型路由修复
  • 修复手动选择 Kimi 等模型后，普通文字消息仍被自动切换到本地演示模型的问题。
  • 自动切换只用于图片任务；文字任务始终使用底部当前选择的供应商和模型。
  • 适配 Kimi K2.6/K2.5 固定采样参数，避免测试连接和聊天请求因 temperature 报错。
  • Kimi 预设改为国内和 Global 分开，减少 API Key 与 Base URL 混用导致的认证错误。
  • 认证失败提示会显示具体供应商和模型，方便检查对应 API Key 与 Base URL。
neo 0.9.7 打包体验
  • macOS 打包时弱化内部 Helper 的对外显示名，减少 Finder 和 Spotlight 搜索 neo 时的误解。
  • 保留 Electron Helper 的内部文件名和 bundle id，不影响启动、签名和用户工作区数据。
neo 0.9.6 办公附件与界面
  • Office 附件上传限制与办公内核对齐，表格、文档、PDF、PPT 可上传到 25MB。
  • 工具与参数设置增加已保存反馈，本地工具、外部读取、授权路径和参数改动后能看到状态。
  • 主界面整理为 Codex 风格三栏整体区域，左侧对话、中间聊天、右侧工作台各自成区。
  • 关于页增加版本更新内容，后续每次升级都在这里展示具体变化。
neo 0.9.5 办公稳定性
  • 继续走稳定 Chat Completions，不做 Responses API 重构。
  • 新增 Office Task Runtime，文件任务按保存、识别、解析、处理、生成、校验、返回记录步骤。
  • 增强 Excel/CSV、Word/PDF、PPT 基础办公内核，生成文件后会重新读取校验。
  • 修复空工具参数导致的假完成问题，关键工具失败或命中轮次上限时不再显示完成。
