# neo 1.1.0 更新公告

这一版不加新功能，专注把「本地办公 + 信息搜集」两条主线做得更稳。重点是修掉搜索/抓取的脆弱点，以及让流式输出在异常时不再"假装没事"。

## 信息搜集更可靠

- **修复网页中文乱码**：此前抓取网页和搜索结果一律按 UTF-8 解码，国内大量 GBK/GB2312/GB18030 编码的站点会整片乱码。现在按「HTTP 头 charset → BOM → HTML `<meta>` 声明」自动探测编码再解码（Node 自带 ICU，原生支持 gbk/gb18030/big5，无新增依赖）。
- **新增 Bing 国内站兜底，中文优先**：搜索引擎链路在 DuckDuckGo / Brave 之外加入 `cn.bing.com`。含中文的查询优先走国内可直连、中文命中更好的 Bing，英文查询仍优先 DuckDuckGo，互不耽误。Bing 的 `ck/a` 跳转链接会被还原成真实地址。
- **搜索缓存 + 重试**：相同查询在 5 分钟内直接命中缓存，不再重复发起请求；瞬时失败（限流、5xx）会退避后自动重试一次，但超时/中断不重试，避免再白等一个完整超时。
- **抓取改用真实浏览器标识**：降低被搜索引擎识别为爬虫直接返回空页或验证页的概率。

## 流式输出更稳

- **空闲超时**：模型响应头到达后，如果正文连续 45 秒没有任何新数据（上游中途卡死），会自动停止并提示，而不是一直挂着等用户手动停。
- **流内错误如实透出**：部分供应商会在流中途用一条 error 数据行报错（余额不足、限流、模型不存在等）。此前这些会被静默忽略、最终只给一个空回复；现在会把真实报错抛给用户。对「自带 API Key」的使用方式尤其有用。

## 办公文件校验

- **空表判失败**：`verify_office_file` 之前的行数判断是恒真条件（形同虚设），现在要求至少有一行可回读数据，空/损坏表格会被判校验失败。
- **特殊字符转义回归测试**：为生成 Word / PPT / Excel 时的 XML 转义（`&`、`<`、`>` 等）补充回归测试，防止以后改动产生损坏文件。

## 安全与上下文兜底

- **内网 / SSRF 守护**：网页读取和下载在抓取前会校验目标地址，拒绝指向本地、私有网段（10/172.16-31/192.168）、环回（127/::1）和云元数据（169.254.169.254）的请求；域名会先解析再判定，挡住指向内网的域名。
- **服务端历史兜底截断**：除前端的长对话压缩外，服务端再加一道保险——历史字符量超过上限时，保留全部 system 指令和最近若干轮，丢弃中间较早消息并插入省略提示，避免超长对话（含定时任务、子智能体路径）直接打到上游报错。
