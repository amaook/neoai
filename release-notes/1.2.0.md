# neo 1.2.0 更新公告

1.2 聚焦「信息搜集闭环」：把零散的"搜一下、读一篇"升级成"搜集成一份带来源、可落地工作区的简报"。本版为第一批（核心闭环），建立在 1.1 已加固的搜索（Bing 中文优先、缓存、编码、SSRF）之上，仍坚持轻量、不做向量库 / RAG。

## 信息搜集技能

- 新增「信息搜集」技能（设置 → 技能库，默认关闭）。开启后子智能体会按固定套路工作：多轮检索 → 批量读取来源 → 去重归纳 → 写成带来源引用的 Markdown 简报，落地到工作区 `Research/<日期>-<主题>.md`，结尾用「## 来源」列出去重后的标题与 URL。
- 需要同时授权「访问网页」和「写入工作区文件」。

## 批量读网页

- 新增 `read_web_pages` 工具：一次并行读取多个网页（最多 8 个）的正文，把多源搜集从"逐个读"压成一次调用，避免触达工具调用轮次上限。
- 复用 1.1 的编码探测、内网 / SSRF 守护与空闲超时；单个来源失败不影响其它来源。

## 网页正文提取升级

- 网页读取（含单页 `read_web_page` 与批量 `read_web_pages`）改为优先提取 `<article>` / `<main>` 主内容，剥离导航、页眉、页脚、侧栏、表单等模板噪音；提取不到实质内容时回退去噪后的全文。
- 效果：简报更干净、更聚焦，也更省模型上下文。

## 搜索精度参数

- `search_web` 新增 `site`（限定单一站点 / 域名，如 `gov.cn`，通过查询语法对所有引擎生效）和 `freshness`（`day`/`week`/`month`/`year` 时效过滤，best-effort 映射到 DuckDuckGo `df`、Bing `filters`，不支持的引擎自动忽略）。
- 缓存键已纳入站点与时效，避免不同筛选条件互相串结果。

## 带来源的结构化简报

- 新增 `save_research_report` 工具：传入 `title` / `summary` / `sections` / `sources`，自动渲染成统一格式的 Markdown（标题、整理日期、摘要、分主题正文、**去重且带取用日期的来源列表**），默认落到工作区 `Research/<日期>-<主题>.md`。
- 「信息搜集」技能改为优先用它产出报告，让引用格式和结构稳定一致，不再依赖模型每次手写。

## 更顺滑的流式输出

- 重写聊天的流式渲染：把突发到达的网络分片先缓冲，再以稳定节奏逐字揭示，并在生成中的文字末尾显示打字光标，告别"一跳一跳"和整段重排带来的卡顿。
- 渲染节流到约 30fps，不再每收到一个分片就全量重渲染整段 Markdown；正在阅读历史（已向上滚动）时不再被强行拽到底部。
- 驱动改用定时器而非 `requestAnimationFrame`，窗口切到后台时流式也能继续平滑推进，而不是憋到最后一次性蹦出来。

## 验证
