# neo 1.4.0 更新公告

这一版给 neo 加上「看屏幕」的能力——在严格授权下，让模型能截取当前屏幕画面来读取界面、定位元素、给操作指导。这是只读能力：它能看，但不能点。

## 看屏幕技能

- 新增「看屏幕」技能（设置 → 技能库，默认关闭、谨慎开启）。开启后模型可用 `screen_capture` 截取当前（光标所在）显示器的画面，用来：
  - 读取界面上的文字 / 状态
  - 定位某个按钮、菜单、选项的位置
  - 一步步指导你怎么操作
- 截图会自动作为图片提供给模型，看不清可再截一次。

## 新工具 screen_capture

- 在 Electron 主进程用 `desktopCapturer` + `nativeImage` 截屏，可按 `max_width` 缩放以控制体积与 token。
- 截图存入工作区 `Screenshots/`，同时作为图片内容块注入到下一轮对话——OpenAI 兼容 / Anthropic / Gemini 三种协议都支持。
- 历史里只保留最近一张截图，避免每轮重复发送大图。

## 安全与前提（重点）

- **独立权限 `screenView`**：默认关闭，单独授权；授权时会明确提示「屏幕画面会发送给所选模型供应商，离开本机」。
- **只读**：只能截屏查看，**不能点击或操作软件**（GUI 点击属于更后续、风险更高的阶段）。
- **仅桌面端**：浏览器预览模式没有截屏能力，工具会直接提示。
- **需要视觉模型**：当前模型必须支持图片输入（DeepSeek 纯文本模型不行）；不支持时会退回文字说明并提示切换模型。
- **macOS 屏幕录制权限**：首次使用需在「系统设置 → 隐私与安全性 → 屏幕录制」中允许 neo，未授权时工具会给出引导。

## 修复

- 修复 Anthropic（Claude）多轮工具调用的一个隐藏 bug：此前从第二轮起，assistant 的 `tool_use` 块与 user 的 `tool_result` 块在转换时会被降级成 JSON 文本，导致 Claude 拿不到结构化工具结果、多步工具链失真。现在这两类原生块会原样保留。

## 验证

- `npm test` 通过，共 18 个测试文件 171 项测试（新增 `tests/screen-capture.test.mjs` 与 `tests/anthropic-tooluse.test.mjs`）。
- `npm run smoke:office` 通过（`failed: 0`）；`node --check` 校验改动的 server / electron / 前端文件无语法错误。
- 真实截屏与视觉往返需在桌面应用 + 视觉模型下真机验收（单测以 mock 覆盖逻辑）。

## 后续

- 「看屏幕」之后若要「能点界面」（GUI 自动化：点击/输入/滚动），属于更重的阶段，需要原生输入注入与更强的安全约束，单独评估。
