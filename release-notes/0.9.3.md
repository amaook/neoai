# neo 0.9.3 更新公告

本次更新聚焦对话体验、模型选择和 OpenAI 官方增强模式，让 neo 更接近 Codex-like 本地 agent 工作台的完整闭环。

## 主要更新

- 修复助手回复中的 Markdown 标记裸显示问题，`**加粗**`、列表、标题、代码块等内容会按正常排版显示。
- 优化默认助手语气，普通聊天更像自然对话，减少报告腔、模板化标题和过度编号。
- 恢复阿里云百炼/Qwen 主流模型预设，补回 Qwen 3.7、3.6、VL、Omni、Coder 等模型。
- 修复图片模型列表过滤过严的问题，Qwen 3.7 Max/Plus 等支持图片任务的模型会正常出现在图片模型下拉框。
- 新增 agent 流程测试，覆盖用户请求、模型工具调用、权限判断、工具结果回传、最终回复和 SSE 流式事件闭环。
- 新增自恢复执行能力，工具步骤失败后会把失败原因和重规划提示回传给模型，引导它换路径、换参数或换工具继续尝试。
- 新增 OpenAI Responses API 第一阶段支持，OpenAI 官方供应商可切换到 Responses API 增强模式，普通兼容供应商继续使用 Chat Completions。
- 新增 Responses API 工具调用闭环，支持 function_call、function_call_output、本地工具执行和权限失败回传。
