# neo 1.6.0 更新公告

让 neo 真正"懂你的文件夹"：新增**「问我的文件」本地知识检索**——你不用再手动指明答案在哪个文件，neo 会跨整个工作区找出来。这是把 neo 从"文件工具"推向"懂你资料的助手"的关键一步，也是「本地办公更擅长」的最大杠杆。

## 问我的文件（search_workspace）

- 新增 `search_workspace` 工具：跨整个工作区按关键词检索，**能搜进 Office 文件的内容**——Excel / Word / PDF / PPT，而不只是纯文本（旧的 `search_files` 基于 ripgrep，搜不进这些二进制 Office 文件）。
- 返回最相关的文件清单 + 命中片段；「本地文件助手」技能会据此**先检索 → 再读取最相关文件 → 带出处回答**。
- 典型用法："我那份报价单里 X 客户的价格是多少""把几个文档里关于 Y 的内容汇总给我"。

## 知识库做厚（检索排序优化 + 工作区记忆隔离）

- **检索排序升级**：`search_workspace` 打分从"子串总次数"升级为综合相关性——
  - 词频用 log 阻尼并按"覆盖了多少种关键词"打折，**长文档不再靠反复堆同一个词霸榜**；
  - **命中文件名/路径**强加权（最强相关信号）、**命中标题/首段**加权、**正文出现完整短语**（含中文无空格相邻）加权；
  - 结果片段围绕**最佳命中位置**截取、对命中词做 `【】` 高亮，命中分散时最多返回两段；返回结构向后兼容（仅增 `matchedTerms`/`nameMatch`/`snippets` 字段）。
- **记忆按工作区隔离**：标记为「当前工作区」的长期记忆只在创建它的那个工作区里参与注入；换工作区不再串记忆，未选工作区时不注入任何工作区记忆。记忆列表对不属于当前工作区的条目标出「其他工作区」。
- **向量检索**：本批暂不做（路线图标注"可选"），待关键词检索打磨稳定后再单独评估可选的 embeddings 重排（默认关、配了 embeddings 模型才生效）。

## 实现与稳定性

- **关键词检索，本地完成**：遍历工作区（排除 node_modules/隐藏目录，限深度与数量），复用已有的 Office 读取器提取文本再匹配、打分、取片段。
- **无新依赖、不依赖额外模型、无需联网**——稳，且隐私不出本机。
- 纯新增 / 改单函数，不改动现有对话主循环；记忆隔离为前端纯增逻辑。

## 验证

- `npm test` 通过，共 23 个测试文件 **202** 项测试（`tests/search-workspace.test.mjs` 扩充：文本命中、**搜进真实 docx 内容**、无命中不报错、缺参、权限门控，新增**文件名命中排序、完整短语加权、长文档不霸榜、片段高亮**）。
- `npm run smoke:office` 通过（`failed: 0`）；`node --check` 校验改动文件无语法错误；记忆隔离与「其他工作区」标注已在本地预览中实测。

## 关于方向

本版起，neo 的定位主轴正式确定为「**带护栏的本地自主电脑代理（办公最强）**」：先把"有工具、能做稳"的本地文件/办公做厚，再逐步建自主引擎与护栏。详见 DESIGN「产品定位」。
