# neo 1.0.0 更新公告

neo 的第一个正式版。这一版聚焦三件事：界面主题系统、Windows 窗口体验对齐 mac，以及"开箱即用"的环境安装体验。

## 浅色主题

- 设置 → 外观新增"浅色主题"开关，沉浸式界面可在深色 / 浅色之间一键切换，即时生效并自动保存。
- 主界面、历史侧栏、工作台、设置面板和电光边框、星尘粒子等 canvas 动效全部随主题换色。
- 浅色主题动效采用"晨雾靛紫"配色（紫 #6A3FD0 + 蓝 #2F62D8），与浅色面板的 glow UI 同源，整体冷暖一致。
- 实现层面把界面上 600 余处硬编码颜色收敛为 `--lens-*` / `--glowui-*` 两组 CSS 变量色板，浅色主题只是一层变量覆盖；后续增加新主题（如跟随系统）成本接近于零。

## Windows 窗口体验对齐 mac

- 去掉 Windows 上的系统标题栏和菜单栏，窗口改为透明无边框，圆角舱体直接悬浮在桌面上，与 mac 一致。
- 右上角内置自绘的最小化 / 最大化 / 关闭按钮，关闭行为与系统按钮一致（隐藏到托盘）。
- 顶部区域仍可拖动窗口。

## 桌宠默认关闭

- 桌宠改为默认关闭，不再随应用自动出现；需要时在设置 → 桌宠里打开"启用桌宠"。
- 托盘菜单的"显示 / 隐藏桌宠"与该开关保持同步，并跨重启记住状态。

## 一键安装环境

- 安装自动感知：点击"一键补齐环境"后 neo 每 6 秒自动复检并刷新列表，全部装好自动提示完成，不再需要手动点"重新检测"。
- 安装脚本支持单项失败继续执行，结尾汇总失败项；修复 Python 与表格库同时缺失时 `brew install python` 重复执行的问题。
- 修复新版 macOS 上 pip 安装被 PEP 668（externally-managed-environment）拦截导致必败的问题，失败时自动以 `--break-system-packages` 重试。
- 新增"使用国内镜像加速安装"选项：Homebrew 走清华 TUNA 源、pip 走清华 index、npm 走 npmmirror，大陆网络下安装成功率显著提升。

## Windows 一键安装强化

- 修复安装窗口启动命令的 PowerShell 语法错误，此前 Windows 上点击安装可能完全没有反应。
- UAC 提权被拒绝时自动降级为普通权限窗口继续安装，由安装器按需逐项提权，不再整体失败。
- 老系统缺少 winget 时自动下载官方 msixbundle 静默安装（必要时补装 VCLibs 运行库重试），浏览器手动下载只作为最后兜底。
- 安装完成后自动刷新 neo 进程的 PATH（Windows 读取注册表，macOS 补齐 Homebrew 路径），新装的 Node / Git / Python 立即可被检测和调用，无需重启应用。

## 验证

- `npm test` 通过，当前 9 个测试文件共 103 项测试（新增环境安装脚本生成 14 项）。
- 已在本地预览验证深 / 浅主题切换与持久化、环境检测接口、镜像选项存取和"无需补齐"短路逻辑。
- Windows 安装脚本内容由单元测试与括号配平检查保障；UAC 与 winget 真机流程建议在干净的 Windows 机器上发版前复验。
