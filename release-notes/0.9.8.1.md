# neo 0.9.8.1 更新公告

## 重点修复

- 修复手动选择 Kimi、OpenAI、Claude 等模型后，普通文字消息仍可能被自动切换到本地演示模型的问题。
- 自动切换模型现在只用于图片任务；普通文字任务始终使用底部当前选择的供应商和模型。
- Kimi K2、Qwen 普通文本模型不再默认走图片直传，带图片附件时会退回到文件路径和摘要，减少 OpenAI-compatible 接口报错。
- 适配 Kimi K2.6/K2.5 固定采样参数，避免测试连接和聊天请求因 temperature 报错。
- Kimi 预设改为国内和 Global 分开，减少 API Key 与 Base URL 混用导致的认证错误。
- API 认证失败提示会显示具体供应商和模型，方便用户检查该供应商自己的 API Key 和 Base URL。

## 使用提示

- 朋友在底部选择 `Kimi / kimi-k2.6` 后，发送普通文字消息会直接调用 Kimi，不会再自动走 `neo-mock`。
- 如果仍提示认证失败，请重新保存 Kimi 自己的 API Key，并确认国内 Key 使用 `https://api.moonshot.cn/v1`，Global Key 使用 `https://api.moonshot.ai/v1`。
