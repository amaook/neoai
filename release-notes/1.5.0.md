# neo 1.5.0 更新公告

回到「本地办公」主线，补齐最常被需要的输出格式：**PDF 导出**。这一版起，neo 能把内容一键渲染成真实 PDF 文件。

## PDF 导出

- 新增 `create_pdf_file` 工具：把 **Markdown 或 HTML** 渲染成真实 `.pdf` 文件，并回读校验。
  - 传 `markdown`：快速生成排版规整的文档（标题、列表、加粗、行内代码、段落），自动套用干净的中文友好样式。
  - 传 `html`：完全自定义排版。
- 「文档阅读」技能现在可以生成 **Word / PPT / PDF** 三种文档（默认开启该技能 + 授权"写入工作区文件"即可用）。
- 典型用法："把这份周报导出成 PDF"、"生成一份 PDF 版说明文档"。

## 办公模板库

- 新增 `get_template` 工具：内置「周报 / 会议纪要 / 简历 / 项目计划 / 工作总结」可填充骨架。不带参数时列出全部，支持按 id（如 `weekly-report`）或中文名（如 `会议纪要`）获取。
- 「文档阅读」技能会在做这类常见文档时先取骨架再填充，结构更规整；填好后用 create_pdf_file / create_word_file 直接出文档。

## 读图片 / OCR

- 新增 `read_image_file` 工具：读取工作区里的图片或扫描件（png/jpg/jpeg/webp/gif/bmp），自动作为图片交给视觉模型——可用来 OCR 读字、描述图片、提取信息。
- 复用 1.4 的"图片注入"链路（OpenAI 兼容 / Anthropic / Gemini 通用），图片不进文本历史、只保留最近一张控成本。
- 需使用支持图片输入的模型（如 Qwen-VL / GPT-4o 系 / Claude / Gemini）；文本模型会收到"请切换支持图片的模型"的提示。

## Markdown → Word

- `create_word_file` 新增 `markdown` 参数：直接把一段 Markdown 自动转成 Word（一级标题→文档标题、二级标题→章节、列表→项目符号、表格→真实表格），无需手工拆成结构化字段。
- 与生成 PDF 形成对照：同一份 Markdown 既能出 PDF 也能出 Word。

## 实现与稳定性

- PDF 使用桌面端排版引擎（Electron `printToPDF`）渲染，复用已验证的"隐藏窗口渲染"模式（与导出图片同源），**不引入任何新依赖**，输出质量高。
- Markdown→HTML 转换做了 HTML 转义，避免内容里的 `<`、`&`、`<script>` 破坏排版或注入。
- 纯新增、不改动现有对话/工具流程；浏览器预览模式没有渲染引擎，工具会直接给出"仅桌面端可用"的提示。

## 验证

- `npm run smoke:office` 通过（`failed: 0`）；`node --check` 校验改动的 server / electron / 前端文件无语法错误。
- 真实 PDF 渲染需在桌面应用下验收（单测以 mock 渲染回调覆盖工具逻辑）。

## 小结

1.5「办公补缺」到此完整：**PDF 导出 + 办公模板库 + 读图/OCR + Markdown→Word**，全部压在 1.5.0 一个版本内。
