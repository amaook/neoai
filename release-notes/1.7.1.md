# neo 1.7.1 更新公告

「数据表格智能」主题继续做厚：在 1.7.0 的**数据质量体检**和**表格出图**之后，这一版补上**透视 / 分组汇总**——把"按维度统计"这件高频办公活做成一个工具。

## 透视 / 分组汇总（pivot_table）

- 新增 `pivot_table` 工具：对 `.xlsx/.xlsm/.csv/.tsv` 表格做分组聚合，结果另存为新的 `.xlsx`（或 `.csv`）。
- **分组聚合**：用 `group_by` 指定一个或多个维度列、`values` 指定要汇总的数值列、`agg` 选 `sum / avg / count / min / max`；不指定数值列时自动识别，不指定分组时出总计。
- **交叉透视表**：设 `pivot_column`，行维度 × 该列的不同取值生成矩阵（如"部门 × 月份"汇总金额）。
- 结果默认按汇总值降序，方便一眼看出大头；默认另存到工作区 `pivots/` 目录，进「成果区」可打开/定位。
- 典型场景："按部门按月统计销售额""各地区订单数""按项目汇总成本""每个客户的总消费"——不用手写公式或手动拖透视表。

## 实现与稳定性

- **纯 JS、不依赖 Python、不联网**：复用 1.7.0 的读表（`loadTableRows`）、数字解析（千分位/货币/百分号/全角）和建表（`createExcelWorkbook`）能力。
- 纯新增工具，不改动现有对话主循环；权限 `fileWrite`（写结果文件），无需新增权限位。
- CSV 输出带 UTF-8 BOM，Excel 打开中文不乱码。

## 验证

- `npm test` 通过，共 26 个测试文件 **233** 项测试（新增 `tests/pivot-table.test.mjs` 10 项：分组求和/平均/计数、交叉透视、总计、xlsx/csv 输出与回读、真实 xlsx 透视、缺参/非法 agg 报错、权限门控）。
- `npm run smoke:office` 通过（`failed: 0`）；`node --check` 校验改动文件无语法错误。

## 数据表格智能（1.7.x）小结

至此 1.7 主题三件套齐活：**check_table 体检 → pivot_table 透视/汇总 → chart_from_table 出图**，覆盖"先看数据干不干净 → 按维度算清楚 → 画出来"的完整链路。下一步进入新支柱：**自主引擎 + 护栏（1.8.0）**。
