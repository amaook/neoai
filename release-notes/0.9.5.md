# neo 0.9.5 更新公告

## 重点修复

- 工具参数硬校验：`write_file`、`run_command`、`export_image` 缺少必要参数时会直接返回清晰错误，不再把空路径解析到工作区根目录。
- 写文件保护：`write_file` 不允许写入空路径、`.`、工作区根目录或目录路径，避免出现空参数导致的目录写入错误。
- Excel 读取兜底：`read_excel_file` 先使用现有读取器，失败后自动切换到 Python/openpyxl，并保持 sheet、行列数和预览行等返回结构。
- 工具循环熔断：同一工具连续因为缺少参数失败时，neo 会停止本轮工具循环，并明确提示“模型没有传入必要工具参数”。
- UI 状态修正：命中工具轮次上限、关键工具失败、输出文件未验证时，工作台显示“未完成”或“需要继续”，不再显示“任务完成”。
- 子智能体收敛：`invoke_skill` 子任务内部不再重复暴露 `invoke_skill`，减少无效工具调用轮次。

## 办公稳定性方向

- 0.9.5 继续采用稳定的 Chat Completions 主线，不做 Responses API 重构。
- 当前版本先把文件任务的“真实执行、失败停住、结果校验”补牢，为 Excel、CSV、Word、PDF、PPT 的内置办公内核打底。
- 新增 Office Task Runtime：办公任务固定记录“保存文件、识别类型、解析内容、执行处理、生成结果、校验文件、返回用户”，失败时停在具体步骤。
- 文件导入统一记录：原文件名、保存路径、大小、文件类型、解析状态、是否截断、是否支持处理。
- Excel/CSV 内核增强：支持 .xlsx、.xlsm、.csv、.tsv 读取，返回 sheet/表名、行列数、表头、预览行、空行空列和重复行检查。
- 表格清洗默认另存：默认去空行、去空列、去重、裁剪前后空格，并自动识别金额/日期列做基础规范化。
- Word/PDF/PPT 基础内核：Word 读取正文、段落/字数/待办/表格线索；PDF 读取页数和文字；PPTX 读取页数、每页标题和正文。
- 新增真实 Word/PPT 生成：`create_word_file` 生成 .docx，`create_ppt_file` 生成基础 .pptx，均会生成后回读校验。
- 新增 `inspect_office_file` 和 `verify_office_file`：统一检查和校验 Office 文件，不再只看文件是否非空。
- 办公样例验收：新增 `npm run smoke:office`，覆盖 5 个表格、3 个 Word、3 个 PDF、3 个 PPT 和 2 个异常旧版文件。

## 使用提示

- 如果模型连续传空工具参数，neo 会主动停住并说明原因，用户可以补充文件名、输出路径或重新发起任务。
- Excel 文件遇到读取器兼容问题时，会自动尝试 openpyxl fallback；如果本机缺少 Python/openpyxl，会返回明确的环境提示。
- 文件任务只有在工具真实成功并通过输出文件校验后，才会进入完成状态。
