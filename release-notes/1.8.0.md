# neo 1.8.0 更新公告

新支柱开篇：**带护栏的本地自主代理**。在让 neo 更自主地连跑多步之前，先把"每一步都看得见、可审计"这件护栏做扎实。这一版交付护栏的第一块——**操作日志**，并顺手补上一处联网安全加固。

## 操作日志（本地审计）

- 工作台新增「日志」标签：把 neo 每次调用工具（读写文件、运行命令、联网抓取、生成 Word/Excel/PPT/PDF、看屏幕等）的痕迹按时间倒序列清楚。
- 每条记录包含：**工具名、成功 / 失败、耗时、产出文件路径**，以及少量安全的参数摘要（如路径、命令、查询词）。
- **只记元数据、绝不记录正文内容**：文件内容、网页正文、命令输出、表格数据等一律不入库。日志全部留在本地（桌面端写到应用数据目录，网页/开发态写到工作区 `.neo/` 隐藏目录），**永不外发**，超过 2MB 自动裁剪，随时可一键清空。
- 失败、被拒绝、未授权的尝试同样会被记录，方便回看 neo 到底做过/想做什么。

## 联网安全加固（SSRF 跳转校验）

- 抓取网页时改为**手动跟随跳转**，每一跳的目标都重新做内网 / 本地地址校验，堵住"初始公网链接 → 302 跳转到内网"的绕过；非 http(s) 跳转和过多跳转一律拒绝。

## 稳定性修复

- 修正工具结果的成功判定：`list_files` 等直接返回列表的工具此前会被回执误判为"失败"，导致操作日志把成功的调用记成失败、达上限提示里的成功/失败计数也不准。统一为 `toolResultOk()` 判定，裸数组 / 无 `ok` 字段且无 `error` 的结果一律按成功处理。

## 实现与稳定性

- 后端 `server/operation-log.mjs` 负责落盘 / 读取 / 裁剪 / 清空；接进 `runToolWithReceipt`（每次工具调用追加），路由 `/api/operation-log`（GET 读、DELETE 清空）。
- 前端纯新增一个只读面板，不改动现有对话主循环，无新增权限位。

## 验证

- `npm test` 通过，共 27 个测试文件 **242** 项测试（含 `tests/operation-log.test.mjs` 6 项：日志路径、每次调用都记录、失败也记、只记元数据不记正文、最新在前 + 清空、记录产出路径）。
- `npm run smoke:office` 通过（`failed: 0`）；`node --check` 校验改动文件无语法错误。

## 护栏路线

操作日志是第一块。后续护栏按风险推进：**任务级授权 → 全局急停**，到位后再做 plan→act→观察→自我纠错的自主主循环。
