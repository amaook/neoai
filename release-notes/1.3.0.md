# neo 1.3.0 更新公告

这一版给 neo 加上「用脚本操控系统/软件设置」的能力——在严格的授权与确认前提下，让它能真正帮你改设置，而不只是打开软件。

## 系统与软件设置技能

- 新增「系统与软件设置」技能（设置 → 技能库，默认关闭、谨慎开启）。开启后子智能体会按平台用正确写法：
  - macOS：AppleScript（如 `defaults write`、System Events）
  - Windows：PowerShell（如 `Set-ItemProperty`、`reg`）
  - 通用：Shell 命令
- 它会在每条脚本的 `purpose` 里用一句话说清"改什么、怎么撤销"，优先选可逆、影响最小的做法。

## 新工具 run_automation_script

- 参数：`language`（applescript / powershell / shell）、`script`、`purpose`、`mutating`。
- AppleScript / PowerShell 会写入临时脚本文件再执行，避免引号转义问题；平台不匹配（如在 Windows 用 AppleScript）会直接报错。

## 安全设计（重点）

- **独立权限 `systemControl`**：与「本地命令」分开，默认关闭，可单独授权——你可以只允许改设置、不允许跑任意命令。
- **每条脚本单次确认**：复用本地命令的确认条管道，但确认条会**突出显示人话目的和脚本语言**，脚本本体可见，60 秒未操作自动拒绝。
- **高危脚本兜底拦截**：删根目录/家目录、磁盘格式化、写裸磁盘、关机重启、fork 炸弹、PowerShell 递归强删、删除 HKLM 关键注册表等，**即使已授权也直接拒绝**，不进入确认与执行。
- 与「最小权限/不默认控制电脑」定位一致：这是"按需开、可逆优先、人来把关"的脚本操控，不是无人值守的自动化。
